# OldRepublicProfessionalWebsiteQAAutomationProject
I'm creating a project that will test the Old Republic Professional's main website in different ways by creating test cases and automation scripts in an organized and ordered way.

## Project Overview
Automated UI testing suite for the Old Republic Professional insurance website (https://www.oldrepublicpro.com/)

## Test Objectives
- Verify core navigation functionality
- Validate form submissions and validations
- Ensure all product pages load correctly
- Test cross-browser compatibility
- Verify responsive design elements

## Scope
### In Scope:
- Homepage functionality
- Main navigation menu
- Product pages (Public Companies, Private Companies, Law Firms, Financial Institutions, Crime)
- Contact form
- Footer links
- About/Team pages

### Out of Scope:
- Backend API testing
- Performance testing
- Security testing
- Payment processing (if any)

## Test Environment
- **Browsers**: Chrome, Firefox, Safari (WebKit)
- **Framework**: Playwright with Python
- **Test Runner**: Pytest
- **Reporting**: HTML reports with screenshots

## Test Schedule
- **Phase 1**: Smoke & Navigation Tests (Week 1)
- **Phase 2**: Form Testing (Week 2)
- **Phase 3**: Advanced Scenarios (Week 3)
- **Phase 4**: Cross-browser & Reporting (Week 4)
